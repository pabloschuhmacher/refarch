Quick Tour of Pivotal Web Services

  1. Using the Web Interface Apps manager After you have successfully "pushed" your application you can monitor it visually. Open a new browser window and open the Apps Manager by visiting https://console.run.pivotal.io and login to your account. Navigate to the org and space containing your application.

    Why this is important? The Apps Manager allows your team to monitor and track the state of your application in an easy to use interface.

  2. Scale up your application to multiple instances. Scale up the number of instances using the Apps Manager and see Cloud Foundry automatically deploy new instances of your application. For this application 4 is a good number. Revisit your application in the browser and hit refresh and note that the IP addressa and the instance number are changing indicating you are accessing a different instance of your app.

    From the CLI: cf scale intro2pws -i 4

    Why this is important? This shows how Cloud Foundry automatically handles the routing and load balancing for multiple copies of your application. Multiple instance support is beneficial for handling increased load or improving availability of your application. PWS automatically distributes multiple copies between availability zones which protects your app availability from network or data center issues.

  3. Look at the logs. Using the Apps Manager click on the ""Logs" tab and see the logs from your application. Visit the demo app and load different pages.

    From the CLI: cf logs intro2pws

    Why this is important? This demonstrates how Cloud Foundry aggregates and consolidates the logs from your different instances automatically.

  4. Crash RecoveryOrganize two web browsers side by side. One with the Apps Manager visible and one with the intro app visible. On the home screen of the intro app you will see a link labeled "crash". Click on this link and look at the Apps Manager and see that one of the instances will indicate it is "Starting" Also note that you will see in the "Events" area an entry indicating the application has crashed. You can also visit the "Logs" tab and see what caused the crash. (hint: an uncaught exception).

    Why this is important? Cloud Foundry is continually monitoring the state of your application instances and automatically restarting them in the event of a crash.
